# 데이터로 진단하는 업무지구의 성공과 실패

가천대학교 스마트시티학과 | 스마트시티 이론과 실제 기말과제 | 2026

**배포 시스템**: https://wjdgkdus05.github.io/smartcity-final  
**분석 보고서**: `/report` 폴더 참고

## 분석 개요

수도권 최고의 성공 사례인 **판교테크노밸리**와 조성은 완료되었으나 활성화에 저조한 **청라국제업무단지**를 공공데이터 기반으로 비교분석한다.

| 구분 | 판교테크노밸리 | 청라국제업무단지 |
|---|---|---|
| 위치 | 경기도 성남시 분당구 삼평동 | 인천광역시 서구 청라동 |
| 구역 면적 | 597,304㎡ | 387,896㎡ |
| 구역계 출처 | 판교테크노밸리 공식 배치도 기반 디지타이징 | 청라개발계획안내도(2025) 기반 디지타이징 |
| 핵심역 | 판교역 (신분당선) | 청라국제도시역 (공항철도) |

## 폴더 구조

```
smartcity-final/
├── data/
│   ├── raw/                    # 원본 데이터 (Git 미추적)
│   │   ├── boundaries/         # 지구단위계획구역 SHP
│   │   ├── sgis/               # SGIS 인구·종사자 원본
│   │   ├── buildings/          # 건축물대장 원본
│   │   └── landuse/            # 용도지역 원본
│   └── processed/              # 전처리 완료 데이터
│       ├── boundaries/         # 구역계 GeoJSON
│       └── sgis/               # 집계구 통계 GeoJSON
├── scripts/                    # 데이터 전처리 파이썬 스크립트
├── web/                        # GitHub Pages 시스템
│   ├── index.html
│   ├── js/
│   ├── css/
│   └── data/                   # 웹용 경량화 GeoJSON
└── README.md
```

## 전처리 스크립트

| 스크립트 | 설명 |
|---|---|
| `scripts/extract_boundaries.py` | 지구단위계획 SHP에서 구역계 추출 |
| `scripts/process_sgis.py` | SGIS 집계구 데이터 전처리 |
| `scripts/isochrone.py` | 등시간권 분석 (다익스트라) |
| `scripts/landuse_analysis.py` | 토지이용 분석 |

## AI 활용 내역

본 프로젝트는 Claude (Anthropic)를 활용하여 데이터 전처리 코드 작성, 구역계 추출, 분석 스크립트 개발에 도움을 받았다. 분석의 해석과 결론은 본인이 직접 작성하였으며, AI가 생성한 수치는 원데이터로 검증하였다.
